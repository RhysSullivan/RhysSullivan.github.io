Unreal Engine Blueprints Tips and Tricks

I’m currently working on putting this on my website for easier viewing, for now go to View->Print Layout to disable page breaks

 

1. Collapse loops into functions

Working with loops in blueprints gets messy, fast. To avoid running reroute nodes across your graph, I recommend making a function that relates to the loop it is used with. Whenever you place a for loop in a blueprint, make a corresponding function called something like LOOP_FUNCTIONNAME so you know it’s used for loops, and give it the parameters of an index and an actor reference.

 

 

2. Avoid Using Reroute Nodes

When you’re making your blueprint, reroute nodes seem like a great idea since you don’t have to place down more nodes and it has the illusion of making the code more organized. The biggest issue that exists with reroute nodes is that it makes the code not readable on the screen. While this may not seem like a big deal when you’re in the zone programming, it makes debugging extremely challenging because you have to continuously go back to where the reroute node starts, making it a confusing mess.

I.e: At each one of these conditional checks or adds, you have to go back to the start of the reroute node to figure out what it relates to.

 

 

3. Collapse to Function / Macro

It is extremely easy to select a big group of nodes and collapse them into a function or a macro. Collapsing these nodes into this form makes your code much more readable. As a general rule of thumb, I like to keep my functions short and have the name of the function do exactly what the code says.

 

 

4. Sequences

Sequences are a handy node that allow you to control the execution flow of your blueprint graph. To create this node, hold the S key and left click.

 

 

5. Delays In Loops

Delays do not run when they are in a for loop. A quick fix is to make a custom event, give it an index value as if it is a loop, have it execute it’s functionality, run the delay, increase its value and call itself again.

 

 

6. Local Variables / Function Arguments

Inside of a function there is a local variable tab, if your variable does not need to exist outside of the function, create it locally in the function for better organization. Function arguments are not found in the same section but they can be accessed from the search bar in the event graph. Use the functions’ variables instead of dragging off reroute nodes from the start of the function. Keep in mind, when you add a parameter to a function, you have to compile for that parameter to appear in the search bar.

 

 

7. Optimizing Blueprints

Do not worry about the performance of your blueprint as you write it. First, just write the blueprint in whichever way makes most sense to you. You only need to worry about performance if the blueprint starts causing problems after the first iteration. Getting hung up on performance will slow your iteration process down, instead, learn how to use the profiling tools in Unreal to verify that your code is fine after you write it. Generally, the performance hit in blueprints comes from the wires connecting the nodes, not the nodes themselves.

 

8. What Does Casting Do?

When you cast in a blueprint, it creates a hard asset reference to what you are casting to. When that blueprint is loaded into memory, everything it casts to is also loaded in memory. This can lead to loading far too many assets at once, which is bad practice and can cause performance issues. For example, if you have a player that casts to a gun, all of the assets for the player will be loaded into memory, which is fine, but the gun assets will also be loaded into memory even when it’s not used. If that gun casts to an enemy blueprint, the skeletal mesh for that enemy is then also loaded into memory. It’ll continue on like that until all of the dependencies caused by the casts are resolved. To view these dependencies, use the reference viewer. This is not to say casting is bad and you should avoid it, but need to be aware of what it’s doing.

 

9. Soft Asset References

Using soft asset references is a way to get around the issues caused by hard asset references mentioned in the section about what casting does. With these, you first need to load the asset into memory and then operate on it, see: https://www.youtube.com/watch?v=K0ENnLV19Cw

 

10. Making Variables Private

This is more of a personal opinion, but I believe that you should make your blueprint variables private. To make a variable private there is a little checkbox in the details panel. This prevents that variable from being directly accessed from outside of the blueprint (*this is a simplification, for a better understanding read up on C++ public/private). Access has to be run through a getter and setter method instead. This allows you to easily add in conditions that need to be met when setting variables, for example, instead of decreasing the health variable of the player directly from a bullet hitting them, you decrease it through a setter method. This allows you to add a check for a condition when the variable is set. For example, if you are setting a health variable and it is below 0 it allows you to call a destroy function. Utilizing private variables and getter/setter methods is how traditional object oriented programming is done and results in cleaner code.

 

11. You can drag types onto events and functions to automatically add them as a parameter

Drag the connection area of a pin onto a function / event and it will automatically be added

 

 

12. Hold control to automatically create a “get” reference of a variable when dragging from the details panel

 

13. Hold alt to automatically create a“set” reference of a variable when dragging from the details panel

 

14. You can assign custom categories to functions and variables

In the details panel, under category, type in the category you would like to assign the function or variable to. If the category you typed does not already exist, a new category will be created. This is very useful for code organization.

 

 

15. Hold down “B” to automatically place a branch node

There are other hotkeys that you can use to place different nodes automatically but this is the only one I find myself consistently using.

 

 

16. You can have multiple event graphs for the same blueprint

I personally have never used this feature before but it’s worth noting. This could be used to help organize code.

 

 

17. Blueprint Function Libraries

To easily share functions between blueprints you can make something called a “Blueprint Function Library,” this allows you to create a function inside of this blueprint graph and all of your blueprints will have access to it.

 

18. Variable Validation

Here are the three ways to check if a reference is valid, my personal preference is #3. To reference a validated get, take a get of a reference, right click on the returned value pin and click “Convert to Validated Get.”

 

 

19. Instance Editable / Expose on Spawn

If you want your actor to reference something that is in the game world, make a variable of that actor type and click on “Expose on Spawn” and “Instance Editable.” Expose on spawn essentially makes the variable visible in the details panel when the blueprint is spawned. Instance editable means this is editable per instance. With multiple instances of your actor spawned in the level, you can then select certain actors to be referred to by this variable.

 

 

20. Splitting Structs

A Struct is a “larger” variable containing “smaller” variables. In order to access these “smaller” variables, the struct has to be split in the blueprint. My preferred method for clean code is to drag off of the struct and type in “break,” Unreal Engine will autocomplete what struct type to break, then you can drag off of the pin and proceed with your code. You can also just split the struct pin in place, however this prevents the pin from being collapsed.

 

 

21. Palette

Under the window dropdown, there is a palette that allows you to store frequently used blueprint nodes for quick access.

 

 

 

22. Bookmarks

You can bookmark frequent locations you access in your blueprints. To do this, first bring up the bookmark window by going to the window dropdown and then clicking bookmarks. To add a bookmark, click on the star, add a name, and it will appear in the section. This also stores comment locations

 

23. Debugging

Blueprints support traditional debugging techniques such as watching variables and breakpoints. You can also watch the sequence of your code run real time in the event graph to see which nodes are running.

 

24. Finding References

You can find where different variables, functions, or events are being get or set in your blueprints by right clicking on them in the blueprint tab and clicking “Find References.”

 

25. Hovering over the Functions Section In Blueprints allows you to see all of the functions available to be overridden

 

26. Variables inherited from parent classes can be found by clicking on the eye icon

 

27. You can assign custom events to run when other events are called

29. You can make blueprints save on compile

This is helpful for not losing progress when Unreal crashes.

 

 

30. Split Struct Pin

You can split up the pin of a blueprint function to its base components, allowing you to plug in individual values that are otherwise inaccessible.

 

31. Shift + Delete to delete a node & automatically connect the previous and next nodes

32. Ctrl Click + Drag to grab the end of a wire

 

33. Alt Click on a connection flow wire to disconnect it

34. You can reorder sequence nodes by left clicking on the pin

 

35. Use custom events to clean up your event graph

Instead of having a bunch of crossing wires, make a custom event to call where you need to
